# Workspace

## Overview

pnpm workspace monorepo using TypeScript. This is the **Damumed КМИС** (Казахстан Медициналық Ақпараттық Жүйесі) mockup — a medical information system for the Aqbobek rehabilitation center. Built for the AIS Hack 3.0 HealthTech & RPA track.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)
- **Frontend**: React + Vite + Tailwind CSS + shadcn/ui

## Project Purpose

Mockup of Damumed КМИС for the Aqbobek rehabilitation center that enables:
- Patient admission and record management
- Medical forms (primary exam, daily diary, epicrisis, consultations)
- Automated 9-day rehabilitation schedule generation
- Status/appointment tracking for doctors, physiotherapists, masseurs, psychologists
- Doctor confirmation flow before form submission (safety requirement)

## Artifacts

- **artifacts/damumed** — Frontend React + Vite app at `/` (path: `/`)
- **artifacts/api-server** — Express API server at `/api`

## Authentication

JWT-based authentication for doctor-only access:
- `POST /api/auth/login` — Returns JWT token (12h expiry)
- `GET /api/auth/me` — Get current user from token
- All `/api/patients`, `/api/forms`, `/api/appointments`, `/api/dashboard` routes require `Authorization: Bearer <token>`
- Demo accounts: `asanov`/`doctor123`, `zhumabek`/`doctor123`, `seytkali`/`doctor123`, `admin`/`admin123`
- Frontend: `AuthProvider` in `src/contexts/auth-context.tsx` stores JWT in localStorage, calls `setAuthTokenGetter` for all API client requests
- Backend middleware: `artifacts/api-server/src/middlewares/auth.ts` — `requireAuth` + `requireDoctor`

## Key API Endpoints

- `GET /api/patients` — List patients (search, status filter)
- `POST /api/patients` — Admit new patient
- `GET /api/patients/:id` — Patient detail
- `PATCH /api/patients/:id` — Update patient
- `GET /api/patients/:id/forms` — Patient's medical forms
- `POST /api/forms` — Create medical form
- `GET /api/forms/:id` — Get form
- `PATCH /api/forms/:id` — Update form
- `POST /api/forms/:id/submit` — Submit form (doctor confirmation)
- `GET /api/appointments` — List appointments
- `POST /api/appointments` — Create appointment
- `PATCH /api/appointments/:id` — Update appointment status
- `POST /api/schedule/generate` — Auto-generate 9-day rehabilitation schedule
- `GET /api/dashboard/summary` — Dashboard statistics
- `GET /api/dashboard/today` — Today's appointments

## Database Schema

- `patients` table — patient records
- `medical_forms` table — medical documentation
- `appointments` table — rehabilitation schedule

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

**Note after codegen**: The `lib/api-zod/src/index.ts` must only export `./generated/api` (not `./generated/types` which gets auto-generated by orval). Manually fix this after any codegen run.

See the `pnpm-workspace` skill for workspace structure, TypeScript setup, and package details.
